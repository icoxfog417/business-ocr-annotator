# Proposal: Switch to NVIDIA Nemotron Nano 12B for Annotation Generation

**Date**: 2026-01-11
**Author**: Development Team
**Status**: Proposed

## Background

The original design specified using Amazon Bedrock (Claude 3.5 Sonnet, Qwen-VL) for AI-powered annotation generation. However, license limitations for using annotations generated by these models have been identified, requiring a switch to an open-weight model with more permissive licensing.

## Proposal

Replace Amazon Bedrock integration with NVIDIA Nemotron Nano 12B model for annotation generation:

- **Model**: NVIDIA Nemotron Nano 12B (open-weight, permissive license)
- **Deployment**: Self-hosted or via compatible API endpoint
- **Integration**: Lambda function calls Nemotron API instead of Bedrock
- **Licensing**: Annotations can be used without restrictions

## Impact

### Requirements Changes
- Update REQ-BR-001: Change from "Amazon Bedrock" to "NVIDIA Nemotron Nano 12B"
- Update REQ-BR-002: Remove Bedrock model references, specify Nemotron
- Update REQ-MC-001-005: Adapt model configuration for Nemotron

### Design Changes
- Replace BedrockVisionClient with NemotronVisionClient
- Update Lambda function architecture diagrams
- Modify API integration specifications
- Update environment variables and configuration

### Task Changes
- Replace "Bedrock Setup" tasks with "Nemotron Setup" tasks
- Update Lambda function implementation tasks
- Modify model configuration tasks
- Update testing and validation tasks

## Alternatives Considered

1. **Continue with Bedrock**: Risk of license restrictions on generated annotations
2. **Other open-weight models**: Nemotron Nano 12B offers good balance of performance and licensing
3. **Multiple model support**: Added complexity, can be future enhancement

## Implementation Plan

1. Update specifications (requirements.md, design.md, tasks.md)
2. Research Nemotron Nano 12B deployment options
3. Create sandbox verification for Nemotron integration
4. Update Lambda function implementation
5. Modify frontend integration
6. Update documentation and configuration

## License Benefits

- Generated annotations can be freely used and distributed
- No restrictions on commercial use of datasets
- Compatible with open dataset publishing on Hugging Face
- Aligns with project goal of creating unrestricted OCR datasets

- Need to verify Nemotron Nano 12B vision capabilities
- Determine optimal deployment method (self-hosted vs API)
- Ensure performance meets Sprint 2 requirements
